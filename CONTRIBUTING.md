# Contributing to AI Data Platform 2025

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/AI-Data-Autopilot.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit with clear messages
7. Push to your fork
8. Create a Pull Request

## Development Setup

```bash
# Clone the repository
git clone https://github.com/PatelJU/AI-Data-Autopilot.git
cd AI-Data-Autopilot

# Install dependencies
pip install -r requirements.txt

# Run tests
pytest tests/

# Run the application
python run.py
```

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable names
- Add docstrings to functions and classes
- Keep functions focused and small
- Comment complex logic

## Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Run: `pytest tests/`

## Pull Request Guidelines

1. **Clear Title**: Describe what the PR does
2. **Description**: Explain the changes and why
3. **Tests**: Include tests for new features
4. **Documentation**: Update docs if needed
5. **One Feature**: One PR per feature/fix

## Reporting Issues

When reporting issues, please include:

- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Python version and OS
- Error messages (if any)

## Feature Requests

We welcome feature requests! Please:

- Check if it already exists
- Describe the feature clearly
- Explain the use case
- Suggest implementation (optional)

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Help others learn

## Questions?

Feel free to open an issue for questions or discussions.
